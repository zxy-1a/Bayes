================================================================================
【关键模块说明】
================================================================================

📦 disambiguate_complaint.py
   功能: 处理模糊诉求，生成澄清问题或推断症状
   用途: 当诉求不够具体时使用（如"心烦"、"没劲"、"不舒服"）
   本地规则: VAGUE_COMPLAINT_PATTERNS 字典

📦 recommend_pipeline.py
   功能: 端到端协调所有推荐步骤
   用途: 完整的推荐流程（拆分 → 澄清 → 搜索 → 推荐）

📦 api_interface.py
   功能: 定义前端与后端的交互接口
   用途: WebSocket / REST API 的接口规范

📦 test_pipeline.py
   功能: 综合测试和演示
   用途: 验证系统功能是否正常

📦 demo_workflow.py
   功能: 交互式演示
   用途: 展示三个典型场景

【工作流示例】
================================================================================

用户输入: "最近觉得心烦，卵巢囊肿，最近容易疲劳"

【Step 1】拆分诉求
   ✓ 卵巢囊肿 (明确)
   ✓ 容易疲劳 (明确)
   ⚠ 心烦 (模糊)

【Step 2】对模糊诉求澄清
   系统问: "您的心烦是否伴随失眠？"
   用户答: "是的，经常失眠多梦"
   推断: 失眠 + 心神不宁

【Step 3】合并症状搜索
   搜索词: [卵巢囊肿, 易疲劳, 失眠, 心神不宁]

【Step 4】推荐茶饮
   → 舒安茶 (匹配: 失眠 + 心神不宁)
   → 肝护茶 (匹配: 卵巢囊肿)
   → 元气茶 (匹配: 易疲劳)

【本地规则库扩展】
================================================================================

在 disambiguate_complaint.py 中编辑 VAGUE_COMPLAINT_PATTERNS：

VAGUE_COMPLAINT_PATTERNS = {
    "新的模糊诉求": {
        "keywords": ["同义词1", "同义词2"],
        "clarification_questions": [
            "澄清问题1？",
            "澄清问题2？",
            "澄清问题3？"
        ],
        "response_mapping": {
            "关键词1|关键词2": ["规范症状1", "规范症状2"],
            "关键词3|关键词4": ["规范症状3"]
        },
        "default_normalized": ["默认规范症状1", "默认规范症状2"]
    }
}

【常见问题】
================================================================================

Q: 本地规则无法匹配怎么办？
A: 系统会尝试调用 LLM API（需要配置 DASHSCOPE_API_KEY）

Q: 如何添加新的澄清问题？
A: 编辑 VAGUE_COMPLAINT_PATTERNS 字典，添加新的问题和映射关系

Q: 推荐结果不准确怎么办？
A: 检查 RAG 知识库构建，或调整澄清问题的映射规则

Q: 支持多语言（方言）吗？
A: 目前不支持，但可以在澄清问题中提供标准话提示

Q: 系统的典型延迟是多少？
A: 本地规则命中：< 100ms；需要澄清：< 500ms

================================================================================
【性能优化建议】
================================================================================

1. 本地规则优先
   • VAGUE_COMPLAINT_PATTERNS 覆盖 90% 的常见诉求
   • 避免不必要的 API 调用

2. 缓存策略
   • LLM 结果缓存 5 分钟
   • 常见诉求缓存 1 天

3. 并行处理
   • 多个模糊诉求同时处理
   • 各诉求的澄清问题并行生成

4. 监控指标
   • 本地规则命中率 (目标: > 95%)
   • 澄清成功率 (目标: > 90%)
   • 平均延迟 (目标: < 300ms)
